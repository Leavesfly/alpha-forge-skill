# 定投（定期定额 / DCA）参考

定投（Dollar-Cost Averaging）按**固定周期投入固定金额**，靠在不同价格分批买入
摊薄持仓成本。它与项目中其他「信号择时」策略本质不同：不预测方向、不做满仓/空仓
切换，而是一个**资金注入节奏**策略。因此本项目为它单独实现了现金流账本式回测引擎
（`scripts/dca/`），不复用 `{-1,0,1}` 信号-仓位框架。

除纯定投外，还内置三种**增强定投**模式（`--mode`），并统一与「一次性投入」「纯定投」
两条基准对比，让"增强是否真的更优"可证伪。对应 CLI：[`run_dca.py`](../scripts/run_dca.py)。

## 为什么单独建模

| 维度 | 信号策略（`run_backtest.py`） | 定投（`run_dca.py`） |
| --- | --- | --- |
| 资金 | 单一固定资金池，满仓/空仓切换 | 每期注入新现金，份额累积 |
| 收益口径 | 时间加权（净值 `cumprod`） | **资金加权（XIRR / 内部收益率）** |
| 核心指标 | 夏普、卡玛、最大回撤 | 资金加权年化收益率、成本调整净值回撤 |
| 基准 | Buy & Hold | **一次性投入** + **纯定投**（双基准） |

如果用时间加权收益去衡量定投，因资金分批注入、每笔资金实际持有时长不同，算出的
年化收益率是错的。**XIRR** 对每笔带日期的现金流求解使净现值为 0 的年化贴现率，
才是定投的正确收益度量。

## 五种定投模式（`--mode`）

| 模式 | 机制 | 每期投入 | 会否卖出 |
| --- | --- | --- | --- |
| `fixed` | 纯定投（基准） | 恒定 `amount` | 否 |
| `ma` | 收盘价低于 `ma_window` 均线时投 `boost` 倍（单档） | `amount` 或 `amount×boost` | 否 |
| `smart` | 智能定投：按**偏离均线的幅度**分档，越便宜投越多、越贵越少乃至暂停 | `amount ×` 分档乘数 | 否 |
| `dip` | 超跌回撤加码：按**距近期高点的回撤深度**分档，叠加 RSI 超卖再加一档 | `amount ×` 分档乘数 | 否 |
| `value_avg` | 价值平均：盯住**目标市值增长线**，每期补足差额，涨过目标则卖出 | 目标市值 − 当前市值 | **是** |

### smart（智能定投）分档

以相对均线偏离 `dev = 收盘价 / MA − 1` 分档（默认 `boost=2.0` 时的乘数）：

| 偏离 `dev` | 乘数 | 含义 |
| --- | --- | --- |
| `dev ≤ −15%` | `boost`（2.0） | 大幅低于均线，重仓加码 |
| `−15% < dev ≤ −5%` | `1+(boost−1)/2`（1.5） | 偏低，适度加码 |
| `−5% < dev < 5%` | 1.0 | 均线附近，常额 |
| `5% ≤ dev < 15%` | 0.5 | 偏贵，减额 |
| `dev ≥ 15%` | 0.0 | 太贵，暂停定投 |

### dip（超跌回撤加码）分档

以距 `dip_window` 期滚动高点的回撤 `dd = 收盘价 / 近期高点 − 1` 分档：

| 回撤 `dd` | 乘数（`boost=2.0`） |
| --- | --- |
| `dd > −5%` | 1.0 |
| `dd ≤ −5%` | `1+(boost−1)/2`（1.5） |
| `dd ≤ −15%` | `boost`（2.0） |
| `dd ≤ −30%` | `boost×1.5`（3.0） |

叠加：若 RSI(14) `< 30`（超卖），在上述基础上再 `+0.5`（封顶 `boost×1.5`）。

### value_avg（价值平均）

设第 `k` 个定投期的目标市值 `T_k = amount × k`（每期目标增长 `amount`）。每期计算
`gap = T_k − 当前市值`：`gap > 0` 补仓买入，`gap < 0`（涨过目标）卖出减仓至目标。
这使它机制上比 DCA 更激进地"低买高卖"，但**长牛里会过早减仓、跑输纯定投**——这正是
需要双基准对比的原因。

> ⚠️ 增强模式凡引入择时/加码参数，都可被历史数据优化出漂亮曲线，本质滑回择时与曲线拟合；
> 长期上涨市场里"最优定投"更会平凡地退化为"越早一次性投入越好"。定投的价值在于纪律性与
> 摊薄成本，而非历史最优。参数寻优结果请务必用样本外数据审视。

## 计量方法

### 现金流账本

- 买入以**当日收盘价**成交，成本从买入现金中扣除：`买入份额 = 现金 × (1 − 成本率) / 收盘价`。
- 卖出（仅 `value_avg`）成本从卖出所得中扣除：`到手现金 = 市值 × (1 − 成本率)`。
- `cashflow` 记录投资者视角的实际现金流（买入为负、卖出为正），`contribution` 记录名义交易金额（买入>0、卖出<0）。
- 累计净投入 `invested[t]`、累计份额 `shares[t]`、持仓市值 `market_value[t]` 三条曲线随时间推进。
- 定投日：`monthly` 取每月首个交易日，`weekly` 取每周首个交易日，`daily` 每个交易日；非时间索引退化为固定间隔。

### 资金加权年化收益率（XIRR）

将每笔实际现金流（买入负、卖出正）与期末市值构造带日期序列，用二分法求解年化内部收益率：

```
NPV(r) = Σ  cf_i / (1 + r)^t_i = 0        t_i = (date_i − date_0) / 365
```

实现见 [`dca/metrics.py`](../scripts/dca/metrics.py) 的 `xirr()`。现金流全部同号或缺少日期时返回 `nan`，报告显示 `N/A`。

### 最大回撤

基于**成本调整净值** `市值 / 累计净投入` 的曲线计算（>1 即累计盈利），剔除本金注入影响，比直接对市值曲线求回撤更贴合定投语义。

### 双基准

- **一次性投入**：同等总本金在期初一次性买入并持有到末期，年化为 CAGR。回答"如果一开始就全投进去会怎样"。
- **纯定投**：相同 `amount`、相同周期的 `fixed` 模式（仅增强模式下额外给出），回答"增强相对纯定投是否真的更优"。

## 命令行用法

```bash
cd scripts   # 首次先执行 uv sync

# 每月纯定投 1000（默认），输出定投报告 + 一次性投入基准对比
uv run python run_dca.py --symbol 600000.SH

# 均线加码：收盘价低于 60 日均线时投 2 倍
uv run python run_dca.py --symbol 600519.SH --mode ma --ma-window 60 --boost 2.0 --plot

# 智能定投：按偏离均线幅度分档加码/减码/暂停
uv run python run_dca.py --symbol 600519.SH --mode smart --ma-window 60 --plot

# 超跌回撤加码：按距 120 日高点的回撤深度分档 + RSI 超卖
uv run python run_dca.py --symbol AAPL.US --mode dip --dip-window 120 --count 1000 --plot

# 价值平均：每期目标市值增长 1000（涨过目标会卖出）
uv run python run_dca.py --symbol 600000.SH --mode value_avg --amount 1000 --plot

# 显式分红建模：自动拉 A 股分红历史（不复权价 + 分红再投入）
uv run python run_dca.py --symbol 600000.SH --dividends --div-policy reinvest

# 分红现金落袋（计入 XIRR 现金流）；非 A 股可用 CSV 提供（列：date,dps）
uv run python run_dca.py --symbol 600000.SH --dividends --div-policy cash
uv run python run_dca.py --symbol AAPL.US --dividends my_divs.csv
```

### 主要参数

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| `--symbol` | 必填 | 标的代码，如 `600000.SH` |
| `--mode` | `fixed` | 定投模式：`fixed` / `ma` / `smart` / `dip` / `value_avg` |
| `--freq` | `monthly` | 定投频率：`daily` / `weekly` / `monthly` |
| `--amount` | `1000` | 每期基准投入金额；`value_avg` 下为每期目标市值增量 |
| `--count` | `1250` | 拉取 K 线数量（约 5 年） |
| `--commission` / `--slippage` | `0.0005` | 单边手续费率 / 滑点率 |
| `--ma-window` | `60` | `ma` / `smart` 模式均线窗口 |
| `--boost` | `2.0` | 加码基准倍数（`ma` 低于均线倍数；`smart` / `dip` 分档以此缩放） |
| `--dip-window` | `120` | `dip` 模式回撤参考高点滚动窗口 |
| `--dividends [CSV]` | 关 | 显式分红建模：不带值自动拉 A 股分红历史（akshare），带路径读 CSV（`date,dps`）；启用后自动改用**不复权**价格 |
| `--div-policy` | `reinvest` | 分红处理：`reinvest`（除权日收盘价再投入）/ `cash`（现金落袋，计入 XIRR 现金流与盈亏） |
| `--plot` | 关 | 生成图表 |
| `--output` | 自动 | 图表路径，默认 `../outputs/dca_<标的>_<模式>_<频率>.png` |

### 显式分红建模说明

- 前复权价已**隐含**分红再投效果；显式建模用于看清分红现金流本身（累计分红额、
  落袋策略下的真实 XIRR），因此启用后强制改用不复权价，避免重复计收益。
- 除权日按**当日定投前**的持仓份额计分红（除权日当天买入不享当期分红）；
  除权日非交易日时顺延至其后首个交易日入账，回测区间外的分红丢弃。
- 一次性投入基准同步计入分红（固定份额×每股分红，落袋不再投），保证对比公平；
  报告中「累计现金分红」行仅在有分红到账时显示。

## 输出解读

报告含定投主体 + 基准：

- **定投报告**：定投/交易期数、累计净投入本金、期末市值、累计盈亏、累计收益率、**年化收益率(IRR)**、最大回撤、平均持仓成本。
- **基准 A：纯定投**（增强模式下才有）：同参数 `fixed` 的同口径指标。
- **基准 B：一次性投入**：期初全投的累计收益率与年化 CAGR。

图表三面板（`--plot`）：

1. 持仓市值 vs 累计投入本金（绿色=盈利区、红色=亏损区）。
2. 成本调整净值（市值/投入，>1 即累计盈利）。
3. 价格与定投交易点（橙色▲买入、绿色▼卖出，点越大金额越多）。

> 判定增强是否有效：先比 **IRR vs 基准 A（纯定投）**——高出才说明加码/择时带来正贡献；再比 **基准 B（一次性投入）** 看是否值得分批。

## 编程方式调用

```python
from datafeed import fetch_ohlcv
from dca import run_dca_backtest, format_dca_report, format_lumpsum_report

df = fetch_ohlcv("600519.SH", period="1d", count=800)
result = run_dca_backtest(df, symbol="600519.SH", freq="monthly", amount=1000, mode="smart")
print(format_dca_report(result.metrics))
if result.dca_metrics:                       # 增强模式下的纯定投基准
    print(format_dca_report(result.dca_metrics, title="基准：纯定投"))
print(format_lumpsum_report(result.lumpsum_metrics))
```

## 局限

- 定投不是择时策略，报告不含夏普/胜率等交易型指标（对定投无意义）。
- XIRR 依赖数据带真实日期；免费日 K 满足。非时间索引会返回 `N/A`。
- 未建模分红再投资、税费与最小申购单位；`value_avg` 的成本处理为简化口径（目标市值近似命中）。
