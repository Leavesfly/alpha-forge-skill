# CAN SLIM 检查清单（欧奈尔成长股法则）

`run_canslim.py` 把威廉·欧奈尔《笑傲股市》的 CAN SLIM 七项法则落地为**可核查的纪律检查清单**。
与 `run_score.py` 的四层评分同属纪律过滤层：回答「这只票是否符合 CAN SLIM 成长股画像」，
**不是收益预测**。免费日 K 即可运行；A 股基本面走 akshare 自动获取（无需 API Key），
港美股基本面自动用 yfinance 利润表兜底（财年口径，无需 Key）。

## 七个字母的落地口径

| 字母 | 含义 | 本工具的检查口径 | 阈值（可调） |
|------|------|----------------|-------------|
| **C** | 当季 EPS 增长 | 最近报告期**单季** EPS 同比增速（YTD 累计拆单季；基数非正时「扭亏为盈」视为通过） | ≥ 25%（`--c-growth`） |
| **A** | 年度 EPS 持续增长 | 近 3~4 个完整年度 EPS 复合增速；最新 ROE 达标与否作为质量注记（不单独否决） | ≥ 25%（`--a-growth`）；ROE ≥ 17%（`--roe`） |
| **N** | 新高 | 收盘距 52 周（250 日）高点的距离——买强不买弱 | ≤ 15% |
| **S** | 供求 | 近 50 日上涨日均量 / 下跌日均量（吸筹 vs 派发） | ≥ 1.0 |
| **L** | 龙头 | 加权相对强度（3/6/9/12 个月收益按 0.4/0.2/0.2/0.2 加权，近端加倍）：多标的模式用**横截面 RS 百分位**，单标的退化为**与基准比超额** | 百分位 ≥ 70 / 超额 > 0 |
| **I** | 机构认同 | 机构持仓明细无免费数据源，**诚实标注 unavailable 不猜测**；建议人工核查十大流通股东/基金重仓变化 | — |
| **M** | 市场方向 | 基准收盘站上 MA50 与 MA200 才算确认上行（基准按市场自动选择：510300.SH / 02800.HK / SPY.US，`--benchmark` 可覆盖） | 否决项 |

## 结论纪律（单向降级，利好不救场）

- **M 不满足 → 直接「否」**：欧奈尔纪律为大势不对不买（约 3/4 个股随大盘方向）；
- **失败 ≥ 2 项 → 「否」**；失败 1 项 → 「观察」；
- **C/A 基本面缺失 → 结论封顶「观察」**：没有基本面就不是完整的 CAN SLIM；
- K 线不足 260 根（52 周高点 + 12 个月 RS 无法形成）→ 「无法评分」，不用猜测补齐。

## 快速开始

```bash
cd scripts   # 首次先 uv sync

# 单标的七项详评（A 股自动拉季度 EPS/ROE，免费日 K 即可）
uv run python run_canslim.py --symbol 600519.SH

# 多标的横截面：L 用 RS 百分位（更贴近欧奈尔 RS 评分口径），按通过数/RS 排名
uv run python run_canslim.py --symbols 600519.SH,000858.SZ,300750.SZ,002594.SZ

# 阈值本土化：A 股 25% 高增长稀缺，可放宽（须自知偏离原著标准）
uv run python run_canslim.py --symbol 300750.SZ --c-growth 0.15 --a-growth 0.15

# 港美股：缺省自动用 yfinance 利润表兜底；离线可 CSV 注入（列：period_end,eps[,roe]）
uv run python run_canslim.py --symbol AAPL.US
uv run python run_canslim.py --symbol AAPL.US --fundamentals-csv aapl_eps.csv

# 结构化输出（stdout 纯 JSON，进度转 stderr）
uv run python run_canslim.py --symbol 600519.SH --json > canslim.json
```

## 参数一览

| 参数 | 说明 | 默认 |
|------|------|------|
| `--symbol` / `--symbols` | 单标的详评 / 多标的横截面（二选一） | — |
| `--count` | K 线数量（需 ≥ 260 根才可评） | 600 |
| `--benchmark` | L/M 的基准代码 | 按市场自动选择 |
| `--c-growth` / `--a-growth` | C 单季同比 / A 年度复合增速阈值 | 0.25 / 0.25 |
| `--roe` | A 的 ROE 质量注记阈值 | 0.17 |
| `--no-fundamentals` | 跳过基本面（C/A 标 unavailable，结论封顶「观察」） | 关 |
| `--fundamentals-csv` | 基本面 CSV 注入（仅单标的模式） | — |
| `--json` / `--config` | 结构化输出 / TOML 配置注入 | — |

## 数据口径与防前视

- **EPS 口径**：A 股为报告期累计（YTD）值，C 先拆单季再同比；报告期有缺口（如只有 Q1 与年报）时该期
  不参与拆分，不用多季增量冒充单季；港美股（yfinance）为预拆分的单季/年度摊薄 EPS（财年口径，
  财年末可能非 12 月），直接按同财季同比/财年 CAGR 核查；
- **报告期截断**：晚于最后一根 K 线日期的财报不参与检查（无前视）；
- **自动获取**：A 股走 akshare 财务摘要（东财口径），港美股走 yfinance 利润表
  （季度表仅近 4~5 季，不够同比时 C 诚实标注不可评）；接口异常自动降级为 C/A 不可评
  （stderr 告警），不中断价格类检查。

## 与其他命令的衔接

- 结论「是」的标的 → `run_score.py --symbol <代码>` 做四层纪律评分复核（含交易计划价位与建议仓位）；
- 价格趋势验证 → `run_backtest.py --strategy donchian/momentum`（CAN SLIM 的 N/L 与突破/动量族同源）；
- 批量粗筛 → `run_scan.py`（流动性 + 四层评分）与本命令横截面模式互为补充。

## 局限性（转述时不可省略）

- 阈值为欧奈尔原著对**美股成长股**的统计预设，未经 A 股样本外验证；A 股财报节奏（Q1 与年报密集）、
  涨跌停制度与壳价值扰动均可能削弱适用性；
- I（机构认同）无法程序化核查，七项清单实际最多可评六项；
- 追新高策略回撤大，欧奈尔配套的 7~8% 无条件止损纪律不在本命令内——交易计划请用 `run_score.py`；
- 检查结果不构成投资建议。
